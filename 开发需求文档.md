## 3.5、Redis 缓存层设计（旁路缓存模式）

> 本章为新增章节，描述如何将 Redis 以 **Cache-Aside（旁路缓存）** 模式融入现有四层架构。

### 3.5.1 旁路缓存模式说明

旁路缓存（Cache-Aside / Lazy Loading）是最常用的缓存模式，读写流程如下：

**读操作（Read-Through-Cache-Aside）**：

```
客户端请求
    │
    ▼
Service 层先查 Redis
    │
    ├── 命中（Cache Hit）→ 直接返回，不访问数据库 ✅
    │
    └── 未命中（Cache Miss）→ 查询 PostgreSQL → 写入 Redis → 返回结果
```

**写操作（Write-Invalidate）**：

```
客户端写入（新增/修改/删除）
    │
    ▼
Service 层更新 PostgreSQL（Source of Truth）
    │
    ▼
删除对应 Redis 缓存 Key（而非更新）
    │
    ▼
下次读取时触发 Cache Miss，重新从 DB 加载最新数据
```

> ⚠️ **为什么删除而不是更新缓存？**  
> 并发写场景下，更新缓存可能导致旧数据覆盖新数据（写竞争）；删除是原子操作，更安全。

### 3.5.2 Redis 初始化

```go
package databases

import (
    "context"
    "fmt"
    "log"
    "os"

    "github.com/redis/go-redis/v9"
)

// RDB 全局 Redis 客户端
var RDB *redis.Client

// InitRedis 初始化 Redis 连接
func InitRedis() {
    addr := getEnv("REDIS_ADDR", "localhost:6379")
    password := getEnv("REDIS_PASSWORD", "")
    db := 0 // 使用默认 DB 0

    RDB = redis.NewClient(&redis.Options{
        Addr:     addr,
        Password: password,
        DB:       db,
    })

    // 连通性检查
    ctx := context.Background()
    if _, err := RDB.Ping(ctx).Result(); err != nil {
        log.Fatalf("failed to connect redis: %v", err)
    }
    fmt.Println("Redis connected:", addr)
}
```

### 3.5.3 缓存工具层

**新建文件**: `utils/cache.go`

> 封装通用的 Get/Set/Del 操作，Service 层统一调用，不直接依赖 Redis 客户端。

```go
package utils

import (
    "context"
    "encoding/json"
    "errors"
    "time"

    "gin_demo/databases"
    "github.com/redis/go-redis/v9"
)

// ErrCacheMiss 缓存未命中哨兵错误
var ErrCacheMiss = errors.New("cache miss")

// CacheGet 从 Redis 取值并反序列化到 dest（指针类型）
// 返回 ErrCacheMiss 表示 key 不存在
func CacheGet(ctx context.Context, key string, dest interface{}) error {
    val, err := databases.RDB.Get(ctx, key).Result()
    if errors.Is(err, redis.Nil) {
        return ErrCacheMiss
    }
    if err != nil {
        return err
    }
    return json.Unmarshal([]byte(val), dest)
}

// CacheSet 将 value 序列化后写入 Redis，ttl=0 表示不过期
func CacheSet(ctx context.Context, key string, value interface{}, ttl time.Duration) error {
    data, err := json.Marshal(value)
    if err != nil {
        return err
    }
    return databases.RDB.Set(ctx, key, data, ttl).Err()
}

// CacheDel 删除一个或多个 Key（写操作后使缓存失效）
func CacheDel(ctx context.Context, keys ...string) error {
    return databases.RDB.Del(ctx, keys...).Err()
}

// CacheIncr 对计数类 Key 做原子自增（用于 ViewCount / LikeCount）
func CacheIncr(ctx context.Context, key string) (int64, error) {
    return databases.RDB.Incr(ctx, key).Result()
}
```

### 3.5.4 各业务缓存 Key 规范

统一 Key 命名格式：`{模块}:{资源}:{ID或参数}`，避免 Key 冲突，便于按模式批量清理。

| 缓存 Key 示例 | 数据内容 | TTL | 失效时机 |
|---|---|---|---|
| `post:detail:{id}` | 帖子详情（含作者/分类/标签）| 10 分钟 | 帖子被编辑/删除 |
| `post:list:page:{page}:size:{size}:cat:{catID}` | 帖子分页列表 | 2 分钟 | 任意帖子发布/编辑/删除 |
| `post:view_count:{id}` | 帖子浏览量计数器 | 永久（定期回写）| 定时任务同步到 DB |
| `post:like_count:{id}` | 帖子点赞数计数器 | 永久（定期回写）| 定时任务同步到 DB |
| `category:list` | 分类列表（变动极少）| 1 小时 | 分类增删改 |
| `tag:hot` | 热门标签 Top 50 | 30 分钟 | 标签数据变化 |
| `user:profile:{id}` | 用户公开信息 | 5 分钟 | 用户修改资料 |
| `ai:summary:{post_id}` | 帖子 AI 摘要 | 永久 | 帖子内容被编辑 |
| `session:blacklist:{jti}` | JWT 黑名单（登出用）| Token 剩余有效期 | 自动过期 |

### 3.5.5 帖子列表旁路缓存实现示例

**修改文件**: `services/post_service.go`

```go
func (s *postService) GetPostList(page, pageSize int, categoryID uint) (*PostListResult, error) {
    if page < 1 { page = 1 }
    if pageSize < 1 || pageSize > 50 { pageSize = 20 }

    ctx := context.Background()
    // ── 1. 构造 Cache Key ────────────────────────────────────────────
    cacheKey := fmt.Sprintf("post:list:page:%d:size:%d:cat:%d", page, pageSize, categoryID)

    // ── 2. 尝试从 Redis 读取 ─────────────────────────────────────────
    var result PostListResult
    if err := utils.CacheGet(ctx, cacheKey, &result); err == nil {
        return &result, nil // Cache Hit ✅
    }

    // ── 3. Cache Miss：查询数据库 ────────────────────────────────────
    posts, total, err := s.repo.List(page, pageSize, categoryID)
    if err != nil {
        return nil, err
    }

    // ... 组装 result（同原有逻辑）...

    // ── 4. 写入 Redis，TTL 2 分钟 ────────────────────────────────────
    _ = utils.CacheSet(ctx, cacheKey, result, 2*time.Minute)

    return &result, nil
}
```

**写操作（发帖/编辑/删除）时清除列表缓存**：

```go
func (s *postService) CreatePost(req *CreatePostRequest, authorID uint) (*Post, error) {
    post, err := s.repo.Create(req, authorID)
    if err != nil {
        return nil, err
    }
    // 使所有帖子列表缓存失效（模式删除）
    ctx := context.Background()
    keys, _ := databases.RDB.Keys(ctx, "post:list:*").Result()
    if len(keys) > 0 {
        _ = utils.CacheDel(ctx, keys...)
    }
    return post, nil
}
```

### 3.5.6 帖子详情旁路缓存实现示例

**修改文件**: `services/post_service.go`

```go
func (s *postService) GetPostByID(id uint) (*PostDetail, error) {
    ctx := context.Background()
    cacheKey := fmt.Sprintf("post:detail:%d", id)

    // ── 1. 先查 Redis ────────────────────────────────────────────────
    var detail PostDetail
    if err := utils.CacheGet(ctx, cacheKey, &detail); err == nil {
        // Cache Hit：异步自增浏览量计数器（不阻塞响应）
        go databases.RDB.Incr(ctx, fmt.Sprintf("post:view_count:%d", id))
        return &detail, nil
    }

    // ── 2. Cache Miss：查数据库 ──────────────────────────────────────
    post, err := s.repo.FindByID(id)
    if err != nil {
        return nil, err
    }

    detail = toPostDetail(post) // 转换为 DTO

    // ── 3. 写入缓存，TTL 10 分钟 ─────────────────────────────────────
    _ = utils.CacheSet(ctx, cacheKey, detail, 10*time.Minute)

    // ── 4. 自增浏览量（Redis 计数器，定时同步到 DB）────────────────────
    go databases.RDB.Incr(ctx, fmt.Sprintf("post:view_count:%d", id))

    return &detail, nil
}
```

### 3.5.7 浏览量计数器定时回写

使用 Redis `INCR` 累积浏览量，避免每次请求都写数据库。通过 **定时任务** 批量同步回 PostgreSQL：

**新建文件**: `tasks/sync_view_count.go`

```go
package tasks

import (
    "context"
    "fmt"
    "log"
    "strconv"
    "time"

    "gin_demo/databases"
    "gin_demo/models"
)

// StartViewCountSync 每 5 分钟将 Redis 中的浏览量批量写回数据库
func StartViewCountSync() {
    go func() {
        ticker := time.NewTicker(5 * time.Minute)
        for range ticker.C {
            syncViewCounts()
        }
    }()
}

func syncViewCounts() {
    ctx := context.Background()
    // 扫描所有 post:view_count:* 的 Key
    keys, err := databases.RDB.Keys(ctx, "post:view_count:*").Result()
    if err != nil || len(keys) == 0 {
        return
    }

    for _, key := range keys {
        val, err := databases.RDB.GetDel(ctx, key).Result() // 原子读取并删除
        if err != nil {
            continue
        }
        count, _ := strconv.Atoi(val)
        if count == 0 {
            continue
        }
        // 提取帖子 ID（格式：post:view_count:{id}）
        var postID uint
        fmt.Sscanf(key, "post:view_count:%d", &postID)

        // 原子自增到数据库
        if err := databases.DB.Model(&models.Post{}).
            Where("id = ?", postID).
            UpdateColumn("view_count", gorm.Expr("view_count + ?", count)).Error; err != nil {
            log.Printf("sync view count error postID=%d: %v", postID, err)
        }
    }
}
```

在 `main.go` 中启动定时任务：

```go
func main() {
    databases.InitDatabase()
    databases.InitRedis()
    tasks.StartViewCountSync() // 启动浏览量同步定时任务
    r := router.SetupRouter()
    r.Run(":8080")
}
```

### 3.5.8 JWT 黑名单（登出功能）

```go
// 在 JWT 验证通过后，检查 Token 是否在黑名单中

blackKey := fmt.Sprintf("session:blacklist:%s", jti)

```

**登出接口**（`POST /api/auth/logout`）中将 Token 加入黑名单：

```go
func (ctl *UserController) Logout(c *gin.Context) {
    jti := c.GetString("jti")
    exp := c.GetInt64("exp") // Token 过期时间戳
    ttl := time.Until(time.Unix(exp, 0))
    if ttl > 0 {
        ctx := context.Background()
        blackKey := fmt.Sprintf("session:blacklist:%s", jti)
        databases.RDB.Set(ctx, blackKey, "1", ttl) // TTL = Token 剩余有效期
    }
    utils.Success(c, nil)
}
```

```javascript
// api/ai.js - SSE 流式请求示例
export async function sendChatMessage(sessionId, message, onChunk) {
  const token = localStorage.getItem('auth_token')
  const response = await fetch(`/api/ai/sessions/${sessionId}/chat`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${token}`
    },
    body: JSON.stringify({ content: message })
  })

  const reader = response.body.getReader()
  const decoder = new TextDecoder()
  
  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    const chunk = decoder.decode(value)
    // 解析 SSE: "data: {...}\n\n"
    const lines = chunk.split('\n').filter(l => l.startsWith('data: '))
    for (const line of lines) {
      const data = JSON.parse(line.slice(6))
      onChunk(data.content)
    }
  }
}
```

## 十一、关键注意事项

### 安全

1. **AI 接口必须鉴权** — 防止未登录用户刷 Token 消耗
2. **AI 接口必须限流** — 每用户每分钟最多 N 次请求（使用 `golang.org/x/time/rate`）
3. **内容审核** — 用户发帖/评论需要做基本过滤（敏感词 or 调用 AI 审核）
4. **SQL 注入** — GORM 参数化查询已自动处理，但搜索功能需注意
5. **XSS** — Markdown 渲染需要使用安全的渲染库，避免注入脚本

### 性能

1. **分页必须做** — 帖子列表绝不能全量返回
2. **N+1 查询** — GORM 使用 `Preload` 预加载关联数据
3. **AI 摘要缓存** — 生成后存数据库同时写入 Redis（`ai:summary:{post_id}`），相同帖子不重复调用 AI
4. **图片上传** — 建议对接 OSS（七牛云/阿里云 OSS），不要存本地
5. **Redis 缓存穿透防护** — 对查询不存在的资源（如不存在的帖子 ID），写入空值缓存（TTL 短，如 30 秒），防止请求直穿 DB
6. **Redis 缓存雪崩防护** — 为同类 Key 的 TTL 增加随机抖动（如 `TTL ± rand(30s)`），避免大量 Key 同时失效
7. **Redis 缓存击穿防护** — 热点帖子详情可使用 `singleflight` 合并并发的 DB 请求，防止缓存失效瞬间大量请求同时打到 DB

### 代码规范

1. 每个新模块遵循现有的 **四层架构**：`controller → service → repository → database`
2. 接口方法统一在 `service` 层定义接口，`controller` 只调用接口
3. 错误信息不暴露内部细节（数据库错误等）
