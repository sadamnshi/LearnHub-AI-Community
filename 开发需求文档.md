### 1.3 关键待改进点（开发前必读）

> 这些问题需要在后续迭代中优先处理，否则会成为技术债务

1. **JWT 密钥硬编码** — `"CHANGE_ME_TO_ENV_SECRET"` 同时出现在 `services/user_service.go` 和 `router/router.go`，需要统一读取环境变量
2. **CORS 配置位置错误** — `main.go` 中的 `r.Use(cors.New(...))` 在 `router.SetupRouter()` **之后**执行，CORS 中间件不会生效，应移到路由器内部
3. **User 模型缺字段** — 缺少 `Email`、`Avatar`、`Bio`、`Role` 等扩展字段
4. **无统一错误响应格式** — 各接口返回结构不一致，需要封装统一响应体
5. **无配置文件管理** — 数据库、JWT 密钥、端口等需要通过 `.env` 或 `config.yaml` 统一管理

### 3.4 统一响应封装

**新建文件**: `utils/response.go`

```go
type Response struct {
    Code int         `json:"code"` // 0=成功, 其他=错误码
    Msg  string      `json:"msg"`
    Data interface{} `json:"data,omitempty"`
}

func Success(c *gin.Context, data interface{}) {
    c.JSON(200, Response{Code: 0, Msg: "success", Data: data})
}

func Fail(c *gin.Context, httpStatus int, code int, msg string) {
    c.JSON(httpStatus, Response{Code: code, Msg: msg})
}
```

```javascript
// api/ai.js - SSE 流式请求示例
export async function sendChatMessage(sessionId, message, onChunk) {
  const token = localStorage.getItem('auth_token')
  const response = await fetch(`/api/ai/sessions/${sessionId}/chat`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${token}`
    },
    body: JSON.stringify({ content: message })
  })

  const reader = response.body.getReader()
  const decoder = new TextDecoder()
  
  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    const chunk = decoder.decode(value)
    // 解析 SSE: "data: {...}\n\n"
    const lines = chunk.split('\n').filter(l => l.startsWith('data: '))
    for (const line of lines) {
      const data = JSON.parse(line.slice(6))
      onChunk(data.content)
    }
  }
}
```

## 十一、关键注意事项

### 安全

1. **AI 接口必须鉴权** — 防止未登录用户刷 Token 消耗
2. **AI 接口必须限流** — 每用户每分钟最多 N 次请求（使用 `golang.org/x/time/rate`）
3. **内容审核** — 用户发帖/评论需要做基本过滤（敏感词 or 调用 AI 审核）
4. **SQL 注入** — GORM 参数化查询已自动处理，但搜索功能需注意
5. **XSS** — Markdown 渲染需要使用安全的渲染库，避免注入脚本

### 性能

1. **分页必须做** — 帖子列表绝不能全量返回
2. **N+1 查询** — GORM 使用 `Preload` 预加载关联数据
3. **AI 摘要缓存** — 生成后存数据库，相同帖子不重复调用 AI
4. **图片上传** — 建议对接 OSS（七牛云/阿里云 OSS），不要存本地

### 代码规范

1. 每个新模块遵循现有的 **四层架构**：`controller → service → repository → database`
2. 接口方法统一在 `service` 层定义接口，`controller` 只调用接口
3. 错误信息不暴露内部细节（数据库错误等）
